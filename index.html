1. Introduo

Este trabalho aplica tcnicas de Machine Learning para anlise preditiva do dataset NASA Airfoil Self-Noise, que contm medies experimentais de rudo aerodinmico em perfis de asa. O objetivo principal desenvolver um modelo de regresso supervisionada capaz de prever com preciso os nveis de presso sonora (em decibis) com base em parmetros aerodinmicos crticos:

  • Variveis preditoras:
    Frequncia (frequency)
    ngulo de ataque (attack-angle)
    Comprimento da corda (chord-length)
    Velocidade do fluxo livre (free-stream-velocity)
    Espessura da camada limite (suction-side-displacement-thickness)

  • Varivel alvo:
    scaled-sound-pressure (presso sonora escalonada)


Metodologia

  1. Pr-processamento:

    • Normalizao com StandardScaler
    • Reduo de dimensionalidade com PCA

  2. Modelagem:

    • Algoritmo: RandomForestRegressor
    • Validao: 75%/25% (treino/teste)

  3. Avaliao:

    • Mtricas: Coeficiente R e MSE
    • Anlise de importncia de features

Relevncia e Contexto Acadmico

Este trabalho utiliza dados didticos fornecidos pelo professor e tem como foco a aplicao de tcnicas de Machine Learning. Os resultados obtidos podem contribuir para o desenvolvimento de aeroflios mais silenciosos, com potencial de aplicao nas reas de aeronutica e engenharia.

# 4. Criao e Treinamento da Pipeline
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.ensemble import RandomForestRegressor

# Definio da pipeline
pipeline = Pipeline([
    ('scaler', StandardScaler()),          # Etapa 1: Normalizao
    ('pca', PCA(n_components=0.95)),       # Etapa 2: PCA (95% da varincia)
    ('regressor', RandomForestRegressor(n_estimators=100, random_state=42))  # Etapa 3: Modelo
])

# Treinamento da pipeline
print(" Treinando pipeline...")
pipeline.fit(X_train, y_train)

# Informaes sobre o PCA
pca_component = pipeline.named_steps['pca']
print(f"\n Pipeline treinado com sucesso!")
print(f"\n Anlise PCA:")
for i, var in enumerate(pca_component.explained_variance_ratio_, 1):
    print(f"Componente {i}: {var*100:.1f}% da varincia")
print(f"Total de varincia preservada: {sum(pca_component.explained_variance_ratio_)*100:.0f}%")
print(f"Dimenses reduzidas: {len(X_train.columns)}  {pca_component.n_components_}")

8.3 Matriz de Correlao

Como Interpretar Esta Matriz:

1. O que correlao?

Mede como uma varivel se move em relao outra, variando de -1 a 1.

2. O que mostra?

  • Vermelho (+1 a +0.7): Quando uma varivel aumenta, a outra tambm aumenta
  • Azul (-1 a -0.7): Quando uma varivel aumenta, a outra diminui
  • Branco (~0): Pouca ou nenhuma relao

3. Como usar?

  1. Identifique sua varivel alvo (ex: scaled-sound-pressure)
  2. Procure clulas com:
    • Cores fortes (vermelho/azul intenso)
    • Borda tracejada (|r| 0.7) - indicam relaes estatisticamente significativas
  3. Ignore valores prximos de zero

Por que so relevantes?

As correlaes fortes (com borda tracejada) ajudam a:

  • Identificar as melhores variveis para prever seu alvo
  • Evitar usar variveis irrelevantes no modelo
  • Entender quais fatores influenciam mais o resultado

9. Definio Explcita do Problema (Contedo Adicional)

Problema:

Prever a presso sonora (em decibis) de aeroflios com base em medies aerodinmicas obtidas em tnel de vento.

Motivo:

O dataset contm:

  • 5 variveis de entrada (frequncia, ngulo de ataque, etc.)
  • 1 varivel de sada contnua (scaled-sound-pressure)
  • Relao fsica comprovada entre as variveis e o rudo gerado

Soluo Proposta:

  1. Tipo de Problema: Regresso supervisionada

    • Justificativa: A sada um valor numrico contnuo (dB)

  2. Abordagem:

    • Pr-processamento: Normalizao (StandardScaler) + PCA
    • Modelagem: RandomForestRegressor
    • Avaliao: Mtricas R e MSE na base de teste (25% dos dados)

  3. Objetivo Final:

    • Obter um modelo com R > 0.85 e erro mdio < 5dB
    • Identificar quais variveis aerodinmicas impactam mais no rudo

Interpretao do Grfico e Mtricas

O que estou vendo?
Este grfico compara as previses do modelo (eixo Y) com os valores reais medidos em laboratrio (eixo X) para os 25% dos dados separados para teste.

Como interpretar este grfico:

  • Cada ponto representa uma amostra dos 25% de teste
  • Cor do ponto: Intensidade do erro (azul = pequeno erro, vermelho = grande erro).
  • Linha tracejada: Representa a "previso perfeita" (onde o modelo acertaria exatamente o valor real). Pontos prximos a essa linha indicam boas previses.
  • R (Coeficiente de Determinao): Quo prximo os pontos esto da linha ideal (1.0 = perfeito). Varia de 0% (pssimo) a 100% (perfeito)
  • MSE (Erro Quadrtico Mdio): Mdia dos quadrados dos erros (diferena entre real e predito). Quanto menor, melhor (zero = perfeito).

Interpretao das Mtricas de Avaliao

Como Interpretar as 4 Mtricas:

1. R (Coeficiente de Determinao)

  • Varia de 0 a 1 (0% a 100%)
  • Indica quanto da variao o modelo consegue explicar
  • Interpretao: Quanto mais prximo de 1, melhor

2. MSE (Mean Squared Error)

  • Sempre positivo, quanto menor melhor
  • Penaliza erros grandes mais severamente (eleva ao quadrado)
  • Interpretao: Valores baixos indicam predies precisas

3. RMSE (Root Mean Squared Error)

  • Raiz quadrada do MSE, na mesma unidade dos dados (dB)
  • Mais interpretvel que MSE
  • Interpretao: Representa o erro mdio esperado

4. MAE (Mean Absolute Error)

  • Diferena mdia absoluta entre real e predito
  • Menos sensvel a outliers que RMSE
  • Interpretao: Erro mdio "tpico" do modelo
